## Dash 4 compliance findings

- `app/app.py:13` modern `from dash import Dash, html, dcc, Input, Output, State, ClientsideFunction, DiskcacheManager`. No deprecated namespace imports anywhere in `nanometa_live/` (verified by `grep -rn "dash_html_components\|dash_core_components\|dash_table"` returning empty).
- `app/app.py:167` `suppress_callback_exceptions=True` is hard-coded and defended in a comment block (`app/app.py:157-167`) as covering "dynamic layouts + cache issues." Acceptable but masks whole categories of bugs; tighten by registering the dynamic IDs explicitly where possible.
- 14 callbacks use the legacy list-wrapped form `[Input(...), Input(...)]`/`[State(...)]`. Verified instances: `app/callbacks.py:189,236,259,333,430`; `app/tabs/dashboard_tab.py:547,548,768,1280,1319,1338`; `app/tabs/qc_tab.py:707`; `app/tabs/validation_tab.py:519,758`. Function still works in Dash 4; mechanical fix.
- 0 uses of `Patch()` (`grep -rn "Patch()" app/` empty). All AgGrid `rowData` updates send the full row set every interval tick (e.g. `app/tabs/main_tab.py:301 detailed-organism-table.rowData`, `app/tabs/dashboard_tab.py:679 dashboard-sample-table.rowData`, `app/tabs/qc_tab.py:806 per-sample-table.rowData`, `app/tabs/validation_tab.py:493 blast-stats-table.rowData`). Dash 4 supports `Patch()` for AgGrid row mutation -- low-risk performance win.
- 9 clientside callbacks (5 in `app/app.py` and `app/callbacks.py`, others in `dashboard_tab.py`, `preparation_tab.py`, `config_tab.py`). All registered via `app.clientside_callback(...)` rather than hung-off attribute. No orphan client-side definitions.
- 118 of 172 callbacks set `prevent_initial_call=True`. The remainder are either intentionally driven by intervals or load default content; not a defect by itself.
- Layouts return `html.Div([...])` (verified in `qc_layout.py:35`, `main_layout.py`, etc.). No bare list returns.

## Simplification opportunities (top 5)

1. **Adopt `Patch()` for AgGrid row updates.** Four full-table re-renders fire on every 30 s interval tick (`app/tabs/main_tab.py:301`, `app/tabs/dashboard_tab.py:679`, `app/tabs/qc_tab.py:806`, `app/tabs/validation_tab.py:493`). Switching to `Patch()` for incremental row mutation is the highest-leverage Dash 4 modernization in this codebase.
2. **Consolidate the 24 `Input("update-interval", "n_intervals")` subscribers.** Verified 24 hits in `app/callbacks.py` and the seven tab files. The two cache feeders (`aggregate-kraken-cache`, `per-sample-kraken-cache`, set up in `app/app.py:248-249`) are already in place; route the consumer callbacks through them and add a single dispatcher store that fans out only when the cache fingerprint changes.
3. **Delete or wire `chart_builders.py` (1356 lines) and `export_utils.py` (336 lines).** They survive only because `app/utils/__init__.py:32,52` re-exports them. Either build a real "Export results" dropdown that calls `export_utils` (the README implies one exists) and a "Diversity overview" tab that uses `create_alpha_diversity_chart` etc., or remove these modules and their `__init__` aliases.
4. **Split `dashboard_tab.py` (2303 lines, 31 callbacks) and `preparation_tab.py` (2024 lines, 25 callbacks).** Both already mix verdict-banner, alert-card, supporting-strip, sample-detail, and export concerns in one module. Extract by Zone (1-4 in CLAUDE.md) or by responsibility (alerts vs. metrics vs. export). The dead `update_dashboard_watchlist_panel` block at `dashboard_tab.py:1133` should be deleted in the same pass.
5. **Migrate the 14 list-style callback signatures to comma-separated args.** Mechanical, audit-friendly, and removes a long-tail of confusion when Dash 5 deprecates the legacy form. Verified locations: `app/callbacks.py:189,236,259,333,430`; `app/tabs/dashboard_tab.py:547-548,768,1280,1319,1338`; `app/tabs/qc_tab.py:707`; `app/tabs/validation_tab.py:519,758`.

## Risks

- **`suppress_callback_exceptions=True` is permanent** (`app/app.py:167`). The defense in the comment block is reasonable for dynamic IDs, but it also hides genuine wiring mistakes. The dead `update_dashboard_watchlist_panel` callback survived because nothing complained when its outputs disappeared from the layout.
- **`nanometanf_parser.py` is alive only because tests import it.** If the loader chain in `core/utils/classification_loaders.py` is the real source of truth, deleting the parser would break tests; if the parser is the intended canonical implementation, the loaders should call it. The current state is ambiguous and the parser is 1122 lines.
- **AgGrid full-table re-render every 30 s** can produce visible flicker and lose user state (sort, filter, row selection). `Patch()` solves this; current code does not use it anywhere.
- **Re-exports in `app/utils/__init__.py` and `app/components/__init__.py` mask orphans.** A linter check on "imported but never used outside the package" would flag every entry called out above.
- **`chart_builders.py` `create_classification_donut`** has the same display-name as the unrelated method `_create_classification_donut` in `core/export/report_generator.py:331`. Not a current bug, but a renaming hazard if either side is refactored.
- **Cross-platform conda envs in bundle exports** remain a documented operational risk (CLAUDE.md "Cross-platform restriction" section); not a code defect, but the audit confirms the in-app preparation flow does not surface this constraint to operators outside the docs.
